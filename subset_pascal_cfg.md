# Context-Free Grammar (CFG) for a Subset of Pascal

The grammar below is an **LALR(1)** grammar for a subset of Pascal. It can be modified for recursive-descent parsing by eliminating left recursion. An operator-precedence parser can be constructed for expressions by substituting out for `relop`, `addop`, and `mulop`, and eliminating ε-productions.

---

## A.3 Syntax of a Pascal Subset (CFG Productions)

### Program structure

```
program  →  program id ( identifier_list ) ;
            declarations
            subprogram_declarations
            compound_statement
```

### Identifier list

```
identifier_list  →  id
                 |  identifier_list , id
```

### Declarations

```
declarations  →  declarations var identifier_list : type ;
              |  ε
```

### Type

```
type  →  standard_type
      |  array [ num .. num ] of standard_type
```

### Standard type

```
standard_type  →  integer
               |  real
```

### Subprogram declarations

```
subprogram_declarations  →  subprogram_declarations subprogram_declaration ;
                         |  ε
```

### Subprogram declaration

```
subprogram_declaration  →  subprogram_head declarations compound_statement
```

### Subprogram head

```
subprogram_head  →  function  id arguments : standard_type ;
                 |  procedure id arguments ;
```

### Arguments

```
arguments  →  ( parameter_list )
           |  ε
```

### Parameter list

```
parameter_list  →  identifier_list : type
                |  parameter_list ; identifier_list : type
```

### Compound statement

```
compound_statement  →  begin
                       optional_statements
                       end
```

### Optional statements

```
optional_statements  →  statement_list
                     |  ε
```

### Statement list

```
statement_list  →  statement
                |  statement_list ; statement
```

### Statement

```
statement  →  variable assignop expression
           |  procedure_statement
           |  compound_statement
           |  if expression then statement else statement
           |  while expression do statement
```

> **Note (dangling-else):** Adding the production
> `statement → if expression then statement`
> introduces the classic "dangling-else" ambiguity, which can be resolved as discussed in Section 4.3 (or by shifting on shift/reduce conflicts in Yacc).

### Variable

```
variable  →  id
          |  id [ expression ]
```

### Procedure statement

```
procedure_statement  →  id
                     |  id ( expression_list )
```

### Expression list

```
expression_list  →  expression
                 |  expression_list , expression
```

### Expression

```
expression  →  simple_expression
            |  simple_expression relop simple_expression
```

### Simple expression

```
simple_expression  →  term
                   |  sign term
                   |  simple_expression addop term
```

### Term

```
term  →  factor
      |  term mulop factor
```

### Factor

```
factor  →  id
        |  id ( expression_list )
        |  num
        |  ( expression )
        |  not factor
```

> **Note:** There is no syntactic distinction between a simple variable and a call of a function without parameters. Both are generated by `factor → id`.

### Sign

```
sign  →  +
      |  -
```

## A.4 Lexical Conventions (Token Definitions)

1. Comments are surrounded by `{` and `}` and may not contain a `{`. They may appear after any token.
2. Blanks between tokens are optional, except keywords must be surrounded by blanks, newlines, the beginning of the program, or the final dot.
3. Keywords are reserved and appear in **boldface** in the grammar.

### Identifiers (`id`)

```
letter  →  [a-zA-Z]
digit   →  [0-9]
id      →  letter ( letter | digit )*
```

### Numbers (`num`) — unsigned integers/reals

```
digits             →  digit digit*
optional_fraction  →  . digits | ε
optional_exponent  →  ( E ( + | - | ε ) digits ) | ε
num                →  digits optional_fraction optional_exponent
```

### Operators

| Token      | Lexemes                              |
|------------|--------------------------------------|
| `relop`    | `=`, `<>`, `<`, `<=`, `>=`, `>`  (`<>` denotes ≠) |
| `addop`    | `+`, `-`, `or`                       |
| `mulop`    | `*`, `/`, `div`, `mod`, `and`        |
| `assignop` | `:=`                                 |

---

## Reserved Keywords (boldface terminals)

`program`, `var`, `array`, `of`, `integer`, `real`, `function`, `procedure`,
`begin`, `end`, `if`, `then`, `else`, `while`, `do`, `not`, `or`, `div`, `mod`, `and`.
